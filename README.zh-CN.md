# kanDB

[English](./README.md) | 简体中文

一个面向快速、专注桌面工作流的轻量数据库查看与管理客户端。

> kanDB 目前仍处于早期阶段。这个仓库现在还只是一个启动中的骨架工程，本文档描述的是产品方向与近期路线，而不是已经全部完成的功能列表。

## 概述

kanDB 是一个使用 Rust 和 GPUI 构建的桌面数据库查看与管理客户端。它希望把数据库工作做得更轻量、更安静、更直接，把重点放在查看数据、检查结构，以及完成聚焦的数据操作上，而不是做成一个厚重的一体化 IDE 工具。

## 为什么叫 kanDB

kanDB 中的 `kan` 来自中文“看”。这个名字表达的是产品最核心的理念：数据库工具首先应该帮助用户快速、清晰地看见数据。

## 墨猴

kanDB 选择“墨猴”作为吉祥物和品牌意象。传说中的墨猴体型很小，能帮助主人磨墨、递纸，完成简单的桌面辅助工作。这和 kanDB 的产品定位很契合: 它是一个小巧、克制、不喧宾夺主，但能够切实帮助用户处理数据库桌面任务的工具。

## 产品方向

kanDB 想做的是一个专注型桌面客户端，而不是一个笨重的全能环境。目标是让用户可以快速打开、快速理解，并用它完成常见的数据查看和管理工作，而不被多余的复杂度拖慢。

## 近期路线图

- 本地和远程数据库的连接管理
- schema、数据表和记录浏览
- 面向快速读取与迭代的查询执行体验
- 轻量的记录查看与编辑流程
- 围绕速度和清晰度设计的桌面交互体验

## 技术方向

kanDB 使用 Rust 和 GPUI 构建，目标是在原生性能基础上提供一个足够专注、足够高效的桌面开发者工具体验。

## 当前开发状态

当前仓库还只包含初始 Rust 工程骨架。随着后续实现推进，README 会一起更新。
